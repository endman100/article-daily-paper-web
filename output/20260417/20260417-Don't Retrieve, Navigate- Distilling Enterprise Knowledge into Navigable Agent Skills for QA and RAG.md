# Don't Retrieve, Navigate: Distilling Enterprise Knowledge into Navigable Agent Skills for QA and RAG

Don't Retrieve, Navigate
把企業知識蒸餾成可導航的 agent skill
傳統 RAG 把模型當成被動消費者
丟一批檢索結果進去，希望它能湊出答案
模型根本看不到整個語料怎麼組織、還有什麼沒被檢索到
Corpus2Skill 改成讓代理人主動導航
它會怎麼做到
👇

---

離線階段
系統把文件語料反覆分群
每一層都用 LLM 產生摘要
最後形成一棵可導航的 skill 檔案樹
這棵樹把企業知識結構顯式呈現出來
像是一張地圖，而不是一堆散落的頁面

---

在線服務時
代理人先看到整份語料的鳥瞰圖
再逐步鑽進它認為相關的主題分支
每下探一層就讀到更細的摘要
最後靠 ID 取回完整文件
整個過程像人在導覽手冊裡翻章節

---

這種可視的階層讓代理人具備以前沒有的能力
它可以推理自己應該往哪看
可以從沒有產出的路徑折返
可以跨分支組合線索
這些都是傳統檢索做不到的
檢索是拉一把回來就結束，導航是可以反覆探勘

---

在 WixQA 這個企業客服 RAG 基準上
Corpus2Skill 在所有品質指標上都贏過密集檢索、RAPTOR、以及 agentic RAG 基線
這顯示方法不是只在理論漂亮
在真實企業問題上的增益也很具體
尤其對需要跨文件拼湊答案的題目幫助明顯

---

這篇研究的啟發是檢索典範正在被改寫
未來企業知識系統不是比誰索引快
而是比誰的知識地圖讓代理人容易走
代理人擁有越多結構，就越能做出好的推理
導航式 RAG 可能會成為下一個主流形態
