# UniDoc-RL: A Unified Multi-Task Reinforcement Learning Framework for Document Intelligence

UniDoc-RL
具階層動作與密集獎勵的粗細視覺 RAG 框架
把外部視覺知識接到大型視覺語言模型上聽起來很美，但實際上現行視覺 RAG 常靠通用檢索訊號
那些訊號看不清楚文件裡真正關鍵的細節
UniDoc-RL 用強化學習把檢索與重排序還有主動感知和推理四件事合成一個統一代理人
模型怎麼知道該看哪裡
👇

---

它把視覺資訊獲取視為一個序列決策問題
代理人的動作分成好幾層
粗層次做文件級檢索，往下細化時挑選具體圖片，最後主動裁切感興趣的區域
每一步都在幫模型擺脫無關內容，把注意力聚焦在資訊密集的地方
換句話說它會主動導航到該看的位置而非被迫看完所有資料

---

為了端到端訓練，作者設計了密集多獎勵機制
每個動作都有專屬的任務感知監督，而非共用一個稀疏信號
檢索準不準有一個獎勵，挑圖恰不恰當有一個，裁切有沒有用再一個
多個小獎勵組合起來比單一粗獎勵更好指引訓練
代理人能更快掌握每一步該做什麼

---

訓練用的是 Group Relative Policy Optimization 簡稱 GRPO
這個方法讓多個目標同時被優化，又不需要分離的 value network 來協調
這降低訓練複雜度，規模也更容易擴大
把四個不同的任務放進同一個代理人，工程上需要解決怎麼讓它們互不干擾
這篇論文的貢獻就在這裡

---

數據集這邊也不是隨便挑的
作者整理了一份含細粒度動作標註的高品質推理軌跡資料集
讓訓練的監督訊號更完整，代理人能學到更精細的決策
這類標註資料以前很難拿到
開放出來對後續研究是很實用的基礎

---

在三個基準測試上 UniDoc-RL 都穩定超過既有方法
跟前代 RL 基線相比，最大幅度超過 17 個百分點
這表示分層動作加上密集獎勵的組合確實有效
視覺 RAG 從被動檢索走向主動探索，摸出了一條新路
未來的多模態檢索系統很可能會往這個方向發展
