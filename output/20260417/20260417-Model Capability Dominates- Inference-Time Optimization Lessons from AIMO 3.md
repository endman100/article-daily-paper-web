# Model Capability Dominates: Inference-Time Optimization Lessons from AIMO 3

Model Capability Dominates
AIMO 3 給我們的推論期最佳化教訓
多次取樣再投票，通常能提升數學推理的表現
但當錯誤之間彼此相關時，取樣再多有效樣本數也不會增加多少
一個自然的想法是讓每個投票者用不同的推理策略
這篇論文把這個想法拿到 AIMO 3 競賽上實測
結論卻很出人意料
👇

---

實驗規模很具體
三個模型、二十多組實驗、五十題 IMO 等級問題
一張 H100 80GB、五小時時限
所有 prompt 層級的干預一律失敗
沒有一個設計讓投票準確率超過高溫取樣本身
這不是個別策略運氣不好，而是結構性結果

---

原因其實不難理解
高溫取樣已經在做錯誤去相關這件事
各種 prompt 策略本質上比高溫更弱
降低準確度的幅度比降低錯誤相關性的幅度還大
結果是整體投票分數反而往下掉
這對多數實務工程師是個殘酷提醒

---

作者也量化了模型能力的主導力
在同樣的取樣數下，有八分的能力差距
而 prompt 層級的每一項最佳化都填不平這個差距
模型本身的能力壓倒式決定了上限
推論期的花樣再多，也打不過換一個更強的模型

---

另一個發現值得注意
最佳投票分數與 pass@20 之間仍有差距
但這個差距不是來自 prompt
而是來自選擇損失
也就是在候選中挑出最好答案的能力
解法是加一個驗證器做選擇，而不是更花俏的 prompt

---

這篇研究的教訓非常務實
在推論期投放資源時
第一要務是提升模型本身的能力
第二才是想辦法改善候選選擇
再才是 prompt 工程
排錯順序會浪費大量算力，卻得到幾乎為零的效益
