# TRACER: Trace-Based Adaptive Cost-Efficient Routing for LLM Classification

TRACER
以生產軌跡為基礎的成本自適應 LLM 分類路由
每次呼叫 LLM 做分類時，輸入和輸出其實都被記錄下來
這些紀錄本身就是一份免費、不斷長大的訓練資料
拿它訓練一個輕量代理模型，就能接走一大部分未來流量
邊際推論成本幾乎為零
唯一的問題是：代理什麼時候可靠到能上線
👇

---

TRACER 是一個開源系統，解決的正是這個問題
它用 LLM 的生產軌跡去訓練 ML 代理
再用一個叫 parity gate 的機制決定什麼時候能用
只有當代理與 LLM 的一致率超過使用者指定的門檻時才會被啟用
這讓上線變成一個明確的、可控的決策
不再憑感覺

---

路由邊界透明很重要
TRACER 會產出可解釋的資訊
告訴你代理在哪些區域能接手，哪些區域需要退回給 LLM
也會說明什麼時候代理表現開始打平、為什麼退件
工程師可以直接看到決策邏輯
而不是面對一個黑箱

---

這類設計在成本面的效益非常直接
大量重複、相對簡單的請求被代理吸收
真正需要深度推理的請求才會送到 LLM
隨著資料累積，代理覆蓋範圍會慢慢增加
整體花費自然下降

---

實驗結果很具體
在 77 類意圖分類任務上，根據目標品質不同，代理可以覆蓋大部分流量
在更大的 150 類任務上，代理甚至可以完全取代老師模型
自然語言推斷任務則被 parity gate 擋下來，代理沒有上線
這反而是系統運作正常的證據

---

TRACER 的價值不只是省錢
它讓 LLM 服務的部署有了可被觀察、可被驗證的邊界
哪些流量被 LLM 處理、哪些交給代理，都有依據
這對需要稽核與治理的企業環境特別重要
LLM 的產業落地不會只靠模型本身，還需要這類基礎設施
