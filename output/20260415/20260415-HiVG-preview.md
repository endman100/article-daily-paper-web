可縮放向量圖形（SVG）用程式碼描述圖形，具有解析度無損的特性

近年來，大語言模型開始用自回歸程式合成來生成 SVG，開闢了一個新的研究方向

但現有方法直接沿用自然語言處理的位元組級 token 化，這對幾何結構來說是一個根本性的錯配

數值座標被分解成離散符號，空間關係被打破，導致座標幻覺和低效的超長序列

HiVG 提出了一套專為向量圖形自回歸生成設計的層次化 token 化框架，從根本上解決這個問題 👇

----
HiVG 首先把原始 SVG 字串分解成結構化的原子 token

原子 token 對應 SVG 語法中最基礎的元素，確保語法有效性

接著，可執行的指令與參數組被進一步壓縮成幾何約束的線段 token

線段 token 把相關的座標值和指令聚合在一起，大幅減少序列長度

這個兩層結構同時兼顧了語法正確性和表示效率

----
為了解決空間錯位問題，研究者提出了層次化均值噪音（HMN）初始化策略

HMN 為新的 token 嵌入注入了數值排序信號和語意先驗

這讓模型在初始階段就能理解幾何座標的大致分布，而不是從隨機初始化開始

配合課程訓練範式，模型從簡單 SVG 程式逐步學習到複雜圖形

這種漸進式訓練讓模型對可執行 SVG 程式的學習更穩定

----
在文字轉 SVG 和圖片轉 SVG 兩個任務上，HiVG 都顯著優於傳統 token 化方案

生成保真度提升，生成結果更接近目標圖形的視覺外觀

空間一致性改善，座標幻覺大幅減少，幾何關係更準確

序列效率提升，相同表達能力下需要的 token 數量明顯減少

程式碼已在 GitHub 公開

----
HiVG 的設計理念是讓 token 化方案去適應資料的結構，而不是讓資料去適應通用 NLP 的 token

這個原則在其他需要精確幾何或結構化程式表示的 AI 任務中都有應用潛力

向量圖形生成是 AI 輔助設計工具的重要方向，而 token 化方案是決定其品質上限的關鍵環節

HiVG 展示了領域專用 token 化在視覺程式合成中的巨大潛力
