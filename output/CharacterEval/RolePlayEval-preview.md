你跟 LLM 說請你扮演福爾摩斯，它就真的變成福爾摩斯了嗎？

角色扮演是 LLM 最受歡迎的應用場景之一，但也是最難評估品質的任務之一

問題在於，角色扮演涉及太多層面，從知識到語氣到行為，沒有單一指標能蓋全

一篇針對 LLM 角色扮演能力的系統性評估研究，嘗試建立一套完整的評測方法論

它揭示了當前模型在角色扮演上的真實水準，以及那些被忽略的盲點

👇

----
這項研究首先定義了角色扮演的關鍵面向

角色知識是最基礎的一環，模型必須了解角色的背景故事，人際關係與重要事件

語言風格則是進階要求，每個角色說話的方式，用詞習慣，甚至特定的口頭禪都應該被還原

情感一致性更為關鍵，角色面對不同情境時的情緒反應是否合理

最後是長期記憶能力，在多輪對話中，模型是否會逐漸忘記自己在扮演誰

----
評測方法上，研究團隊採用了多層次的測試策略

基礎層是知識問答，測試模型是否真的認識這個角色

中間層是情境對話，把角色丟進特定場景中，看它的反應是否符合人設

進階層是壓力測試，故意丟出可能讓角色出戲的問題，觀察模型的穩定度

這種分層測試的好處在於，能精準定位模型在哪個層次開始出問題

----
測試涵蓋了多個主流 LLM，包括開源與閉源模型

結果顯示，大多數模型在知識層面的表現都還不錯，能準確回答角色相關的事實性問題

但到了語言風格層面，差距就開始拉大了

部分模型雖然知道角色是誰，卻無法用那個角色的口吻說話

這就像一個演員背熟了台詞，但演技卻讓觀眾完全出戲

----
最令人意外的發現出現在長對話場景中

幾乎所有模型在對話超過一定輪數後，角色一致性都會明顯下降

模型可能在第三輪對話還維持著角色的語氣，到了第十輪就開始用自己的預設口吻回答

這暴露了當前 LLM 在 context window 管理上的根本局限

角色扮演需要的不只是記住角色設定，而是在每一輪對話中持續演出那個角色

----
研究也發現，模型規模並不是決定角色扮演品質的唯一因素

有些較小的模型，如果經過針對性的微調，反而能在特定角色上表現得比大模型更好

這暗示角色扮演能力可能需要專門的訓練策略，而非單純依賴通用能力的提升

想要打造高品質角色扮演應用的開發者，選擇合適的微調方案可能比選擇最大的模型更重要

角色扮演評測的標準化，也將推動這個領域走向更科學，更有依據的發展方向
