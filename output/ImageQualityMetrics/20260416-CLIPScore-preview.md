傳統的圖片生成品質指標幾乎都只關注生成圖片跟真實圖片之間的分布距離

但在 T2I 模型的時代，有一個同樣關鍵的問題被忽略了——生成的圖片到底有沒有忠實反映文字提示的內容

一張畫質極高但跟 prompt 完全無關的圖片，在傳統指標上可能得到很好的分數

CLIPScore 的出現正是為了解決這個文字與影像對齊的評估需求

它利用 CLIP 模型同時理解文字和影像的能力，直接衡量兩者之間的語意吻合程度

這讓 T2I 模型的評估第一次有了專門檢查「有沒有畫對」的工具 👇

----
CLIPScore 的運作原理是利用 CLIP 模型的雙塔架構

CLIP 的文字編碼器會把文字提示轉換成一個向量，影像編碼器會把生成圖片也轉換成一個向量

這兩個向量存在於同一個語意空間中，如果文字描述的內容跟圖片內容一致，兩個向量的方向就會很接近

CLIPScore 計算的就是這兩個向量之間的餘弦相似度

數值越高代表文字和影像的語意越吻合，越低代表兩者之間有落差

這個計算過程非常直覺，不需要任何真實參考圖片，只需要文字提示和生成圖片就能完成評估

----
CLIPScore 之所以能運作，完全仰賴 CLIP 模型本身的跨模態理解能力

CLIP 是用大規模的圖文配對資料訓練出來的，學會了文字和影像之間的語意對應關係

這讓 CLIPScore 天然具備了理解各種複雜描述的潛力

從簡單的「一隻貓坐在桌上」到更抽象的風格描述，CLIPScore 都能給出對應的評分

但 CLIP 的訓練資料有其涵蓋範圍的限制，遇到特殊領域或罕見概念時，評估準確度就會下降

CLIP 也已知會對某些文字模式產生偏好，這些偏差會直接反映在 CLIPScore 的結果上

----
在 CLIPScore 的基礎上，研究社群發展出了多個重要的變體

BLIP-ITC 和 BLIP-ITM 使用 BLIP 模型替代 CLIP，提供了不同的語意理解能力

NegCLIP 加入了負樣本的概念，增強了模型區分語意差異的精細度

MosaiCLIP 則在多個面向上強化了 CLIP 的組合性理解能力

這些變體的出發點都是修正原始 CLIP 在特定場景下的弱點

它們各自擅長不同類型的評估任務，讓研究者可以根據需求選擇最適合的版本

----
CLIPScore 跟 FID 評估的是完全不同的面向

FID 關心的是生成圖片整體看起來像不像真實照片，是品質和真實感的指標

CLIPScore 關心的是生成圖片有沒有畫出 prompt 要求的內容，是對齊度的指標

一張圖片可以在 FID 上表現很好但 CLIPScore 很低，代表品質高但跟指令不符

也可以在 CLIPScore 上表現很好但 FID 很差，代表內容正確但畫質不行

完整的 T2I 評估通常需要同時使用兩類指標，才能涵蓋品質和對齊兩個維度

----
CLIPScore 目前是 T2I 領域中使用最廣泛的文字影像對齊指標

幾乎所有 Diffusion Model 的論文在評估環節都會報告 CLIPScore 或其變體

它的計算速度很快，不需要大量樣本，適合在模型開發過程中頻繁使用

不過 CLIPScore 有一個已知的局限——它對組合性語意的理解不夠細緻

當 prompt 包含多個物件之間的空間關係或屬性綁定時，CLIPScore 容易出現誤判

這也是為什麼後來出現了 VQAScore 和 TIFA 等更細粒度的評估方法，專門針對組合性問題下手
