在 T2I 模型的評估中，物件之間的空間關係一直是最難精確衡量的面向

「貓在桌子上面」和「貓在桌子下面」在語意上只差了一個介詞，但視覺呈現完全不同

多數現有指標對這類空間關係的區分能力都偏弱

VISOR 專門針對視覺語意中的物件關係進行評估，填補了這個領域的空白

它的全名是 Visual Semantic Object Relation，聚焦在生成圖片中物件與物件之間的關係是否正確

這個高度聚焦的評估角度讓它能捕捉到其他通用指標容易漏掉的空間和互動資訊 👇

----
VISOR 評估的核心對象是生成圖片中物件之間的各種關係

空間關係是最基本的類型，包括上下、左右、前後、裡外等相對位置

互動關係涵蓋物件之間的動態交互，例如「人騎腳踏車」「狗追球」

屬性綁定關係則檢查特定屬性是否正確附著在對應的物件上

例如「紅色的帽子和藍色的鞋子」需要確認顏色分別對應到正確的物件

VISOR 會針對每一種關係類型分別計算準確率，讓研究者能精確掌握模型在各種關係上的表現

----
VISOR 的評估方式通常結合了物件偵測和關係驗證兩個步驟

首先辨識出圖片中的所有物件及其位置

然後根據 prompt 中描述的關係來驗證這些物件之間的配置是否正確

這個流程需要同時理解文字描述的語意和圖片中的視覺佈局

不同的 VISOR 實作可能使用不同的物件偵測和關係推理模型

評估的精確度取決於這些底層模型在物件辨識和空間推理上的能力

----
空間關係之所以特別難評估，是因為它需要同時處理多個物件的位置和彼此的相對配置

CLIPScore 使用全局嵌入來比較文圖對齊，這種全局表達方式天生就不擅長編碼局部的空間資訊

T2I 模型本身在生成正確空間關係的時候也經常出錯，這是已知的研究挑戰

一張圖可能把所有正確的物件都生成出來了，但它們之間的相對位置卻完全不對

在這種情況下 CLIPScore 可能仍然給出不錯的分數，因為整體語意是匹配的

VISOR 能精確揪出這類空間配置錯誤，提供更有針對性的評估回饋

VISOR 通常作為專項指標出現在 T2I 模型的評估報告中

它不適合取代 FID 或 CLIPScore 等通用指標，而是作為深度診斷工具來補充整體評估

當研究者特別想了解模型在空間推理能力上的表現時，VISOR 能提供最直接的數據

在組合式場景生成的評估中，VISOR 的價值尤其突出

它幫助推動了 T2I 模型在空間理解能力上的持續改進
