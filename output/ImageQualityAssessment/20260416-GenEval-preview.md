T2I 模型的語意對齊評估長期缺乏對特定視覺元素（物件、屬性、關係）的精確量化手段

CLIPScore 等全域語意指標無法定位生成圖片在哪個具體元素上失敗

評估 T2I 模型的組合推理能力——能否同時生成多個物件並保持正確的屬性和位置——需要結構化的框架

GenEval 設計了一套以物件存在性和屬性為核心的評估框架，使用物件檢測器進行自動量化

這種以物件為中心的評估方式能精確反映模型在「生成說的東西」這一基本能力上的水準

GenEval 已成為 T2I 研究中評估組合生成能力的標準基準之一 👇

----
GenEval 的評估核心是將文字 prompt 解構為可視覺驗證的物件級別斷言

每個 prompt 被分析為包含一個或多個物件的結構化表示，每個物件有其屬性（顏色、數量、位置）

使用預訓練的物件檢測模型（如 DINO、OWL-ViT）在生成圖片中檢測提到的物件

對每個物件斷言計算二元分數（是否成功生成），最後彙整為整體的對齊分數

不同的評估子類（single object、two objects、counting、colors、position、color attribution）分別報告分數

這種細粒度的子類分解使研究者能識別 T2I 模型的具體能力邊界

----
GenEval 基準包含精心設計的 553 個 prompt，覆蓋 6 個核心評估維度

single object 子類測試基本物件生成，two objects 測試多物件共存，counting 測試精確數量控制

colors 子類測試顏色屬性的準確性，position 測試空間關係（左右上下），color attribution 測試屬性綁定

物件檢測器的選擇對 GenEval 的評估精度有重要影響，研究者測試了多個 DINO 版本

prompt 設計採用受控詞彙，避免歧義和過於複雜的語言結構，確保評估的可重現性

GenEval 的自動評估流程完全開源，允許研究者在自己的模型上無縫運行完整評估

----
在 GenEval 基準上，早期 SD 1.5 的整體準確率約為 43%，而 DALL-E 3 達到約 67%

在最困難的「color attribution」（顏色屬性綁定）子類上，即使是最強模型的準確率也僅在 40-50% 左右

「two objects」和「counting」子類上的準確率落差揭示了 T2I 模型在多物件組合上的系統性弱點

FLUX.1 等新世代模型在 GenEval 上的整體得分首次突破 70%，標誌著組合生成能力的重要進步

研究者使用 GenEval 發現了模型在顏色-物件綁定上的偏差：高頻物件-顏色組合的準確率遠高於低頻組合

多個研究在訓練資料中針對性加強 GenEval 弱項，並報告了顯著的分數提升

----
相比 CLIPScore 的全域相似度，GenEval 提供了逐物件的精確計數評估，可解釋性更強

相比 TIFA 的 VQA 問答框架，GenEval 使用物件檢測器進行直接驗證，不依賴語言理解模型

與 T2I-CompBench 的多維度組合評估相比，GenEval 更專注於物件層面，評估更為集中

VQAScore 在問答準確率上與 GenEval 有較高相關性，但兩者的評估細粒度不同

GenEval 的局限在於只評估物件的存在和基本屬性，無法評估藝術風格、美學品質等高層次維度

對於抽象或非物件性的 prompt（如情感、氛圍、抽象藝術），GenEval 的評估框架不適用

GenEval 基準已被 DALL-E 3、SDXL-Turbo、Playground 等主流 T2I 模型的技術報告採用

物件中心評估框架的思路推動了 T2I 研究對組合推理能力的系統性關注

GenEval 的細粒度子類評估揭示了 T2I 模型在屬性綁定和計數上的長期瓶頸，推動了針對性改進

後續工作（如 WISE、T2I-CompBench）在 GenEval 的基礎上擴展了評估的語意類型和難度範圍

作為組合生成評估的奠基性工作，GenEval 的評估設計原則影響了後續多個 T2I 基準的構建
