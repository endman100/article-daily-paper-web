現有 T2I 評估指標主要衡量圖片與 prompt 的表面視覺符合度，忽略了更深層的語意維度

「一個人在雨天撐著傘」——模型可能正確生成了外觀，但傘的物理朝向是否合理？

真實世界中存在大量需要常識和世界知識才能判斷正確性的視覺概念

WISE（World Knowledge-Informed Semantic Evaluation）評估 T2I 模型是否具備世界知識推理能力

這包括：物件的功能屬性（椅子用來坐）、材料屬性（金屬有光澤）、社會文化慣例（婚禮穿白紗）

世界知識評估維度補充了表面視覺符合度評估，為更全面理解 T2I 模型能力提供新視角 👇

----
WISE 評估框架的核心是構建需要世界知識才能正確判斷的評估問題

評估問題類型：功能性問題（物件能否執行其功能）、物質性問題（材料和質感是否符合物理規律）

文化規範問題（場景設定是否符合社會文化慣例）、因果關係問題（場景中的物理因果關係是否合理）

對每個評估問題，WISE 使用具備世界知識的 MLLM 或知識圖譜驗證生成圖片的正確性

世界知識問題與 TIFA 的語意問題的區別：前者需要超越圖片外觀的推理，後者只需視覺感知

WISE 評估集的設計確保問題的答案基於客觀的世界知識，而非主觀偏好

----
WISE 基準資料集構建：從 ConceptNet、WordNet、Google Knowledge Graph 等知識來源提取物件屬性

每個評估條目包含：prompt、預期的世界知識屬性、驗證問題、和人工標注的正確答案

世界知識覆蓋範圍：自然科學知識（動物行為、植物特徵）、工程知識（機械結構）、社會文化知識

MLLM 評判員（GPT-4V）被要求進行 chain-of-thought 推理，先描述視覺內容再應用世界知識

跨文化一致性測試：確認評估問題的正確答案在不同文化背景中具有共識

WISE 提供了 2000 個涵蓋多個知識類別的評估樣本，支持不同 T2I 模型的系統性比較

WISE 揭示了 T2I 模型在世界知識整合上的差距，無法從表面視覺符合度指標（CLIPScore、TIFA）中觀察到

相比 CLIPScore 和 FID，WISE 評估了 T2I 模型更深層的語意能力，兩者可形成互補的評估體系

相比 TIFA，WISE 的問題需要推理而非純粹的感知，評估了更高階的認知能力

相比 VIEScore，WISE 提供了更客觀（基於世界知識）的評估標準，減少了主觀判斷的影響

WISE 的局限性在於世界知識的邊界界定：什麼算世界知識、什麼算主觀偏好，有時難以明確劃分

WISE 評估需要具備強大世界知識的 MLLM（GPT-4V 等），在無 API 訪問的場景部署成本較高

WISE 在跨文化評估場景中需要特別注意：不同文化對「正確」的世界知識理解可能存在差異
