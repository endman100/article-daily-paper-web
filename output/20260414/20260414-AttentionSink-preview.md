Attention Sink in Transformers: A Survey on Utilization, Interpretation, and Mitigation

Transformer 模型裡有一個長期被忽視的現象

少數特定的 token 會吸引模型幾乎所有的注意力，但這些 token 本身並不包含任何有意義的資訊

這個現象叫做 Attention Sink，它在各種規模的 Transformer 模型中普遍存在

它的影響範圍涵蓋訓練穩定性、推理效率，甚至模型產生幻覺的頻率 👇

----
Attention Sink 簡稱 AS，指的是注意力機制把過多的權重集中在少數幾個特定 token 上的現象

這些 token 通常是序列的開頭、標點符號，或其他語意上不重要的位置

在 Transformer 的自注意力計算中，softmax 函數會強制產生一個機率分佈

當模型無法從輸入中找到明確的關注對象時，剩餘的注意力權重就會被傾倒在這些固定位置上

這個現象跨越了不同架構、不同規模，甚至不同訓練方式的 Transformer 模型

----
AS 對模型的影響比表面上看起來更廣

在推理時，被不必要佔用的注意力資源會讓模型在長文本處理時效率下降

在訓練時，AS 的存在會讓模型的注意力分佈難以反映真實的語意結構

KV 快取的記憶體管理也會受到影響，因為 sink token 的 KV 向量必須持續保留

這些問題在長上下文任務中尤其顯著，隨著序列長度增加，AS 的影響也會跟著累積

----
AS 與模型幻覺之間的關係是近年研究的重點之一

當注意力過度集中在無意義的 token 上，模型就缺乏足夠的注意力資源去關注真正關鍵的資訊

這可能導致模型在生成時忽略輸入中的重要細節，進而產生聽起來合理但實際上錯誤的輸出

從解釋模型行為的角度來看，AS 也造成了困擾

當注意力權重不再反映語意重要性時，基於注意力的模型分析方法就會產生誤導性的結論

----
這篇論文是目前第一篇系統性整理 AS 相關研究的綜述

全文圍繞三個核心維度展開：基礎應用、機制解釋、以及策略性緩解

基礎應用部分研究 AS 在不同架構下如何呈現，以及是否能被有效利用

機制解釋部分嘗試從理論與實驗角度回答 AS 為何會產生

策略性緩解部分則整理了各種試圖減少或消除 AS 的方法

----
AS 雖然是一個問題，但研究者也發現了利用它的方式

由於 sink token 的位置通常具有規律性，部分研究把這個特性用在 KV 快取壓縮上

在設計高效推理系統時，確保 sink token 的 KV 向量不被清除，可以維持模型的輸出穩定性

有研究發現，如果強制清除 sink token，模型的輸出品質會急劇下降

這代表 sink token 在模型運作中扮演了某種穩定錨的功能，即使它的語意內容幾乎為零

----
緩解 AS 的策略大致分為三個方向

第一種是在架構層面修改注意力機制，讓注意力分佈更均勻

第二種是引入專門設計的緩衝 token，讓模型有固定的位置可以傾倒多餘的注意力，同時避免干擾真正的語意處理

第三種是透過訓練策略的調整，讓模型在學習過程中自然減少對特定位置的過度依賴

每種方法都在效果、計算成本與模型泛化能力之間存在不同的取捨

AS 的研究橫跨了效率、解釋能力與可靠性三個領域
