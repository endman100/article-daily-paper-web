Efficient RL Training for LLMs with Experience Replay

在強化學習的世界裡，經驗回放（experience replay）是一個非常基礎的技術

它的做法很簡單：把已經收集到的訓練資料存起來，之後重複利用多次

但這個技術幾乎從未被引入大型語言模型的強化學習後訓練流程中

主流觀點認為 LLM 的強化學習訓練必須用「剛剛生成的新鮮資料」才能維持效果，但這篇論文正面挑戰了這個假設 👇

----
在一般的強化學習中，on-policy 和 off-policy 之間的取捨是個基本問題

On-policy 的做法是每次訓練都用目前策略生成的最新資料，確保資料與當前策略完全一致

Off-policy 的做法則是允許使用過去收集的資料，但要處理新舊策略之間的分佈差異

在一般 RL 任務中，生成資料的成本相對低，所以 on-policy 常常是合理的選擇

但在 LLM 訓練中，生成資料的計算成本非常高，這個前提就不再成立了

----
研究者系統性地研究了 replay buffer（回放緩衝區）在 LLM 後訓練中的設計問題

他們把最佳設計公式化為三個因素之間的取捨：資料過時導致的方差增加、樣本多樣性、以及生成資料的計算成本

生成資料（模型推理）是整個訓練流程中計算開銷最大的部分之一

當生成成本夠高時，嚴格堅持 on-policy 採樣在整體效率上就是次優的選擇

適度的 off-policy 設計可以大幅節省這部分的計算資源

----
實驗結果支持了研究者的理論分析

設計良好的 replay buffer 可以大幅減少訓練過程中所需的推理計算量

在多數情況下，這個節省不會對最終模型的表現造成任何損害

在某些設定下，使用 replay buffer 甚至讓最終模型的表現略有提升

研究者把這個現象部分歸因於 replay buffer 帶來的樣本多樣性增加

----
Policy entropy（策略熵）是評估強化學習訓練品質的重要指標之一

它反映了模型輸出的多樣性，過低的策略熵代表模型正在向特定解法過度收斂

研究者發現，使用設計良好的 replay buffer 可以維持甚至改善策略熵

這說明回放本身不只是節省計算的手段，它還對訓練過程的探索性有正向影響

這個發現對於避免 LLM 強化學習訓練中的策略崩塌問題具有實際意義

----
這篇論文的核心貢獻是把一個在一般 RL 領域早已成熟的技術，重新帶回 LLM 訓練的視野中

過去大家認為 on-policy 對 LLM 後訓練是必要條件，但這個研究給出了更細緻的分析

關鍵在於：當資料生成的成本高到一定程度時，適度的「過時容忍度」在效率上是值得的

這個視角為 LLM 強化學習訓練的基礎設施設計提供了一個新的思考框架
