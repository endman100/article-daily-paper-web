SPASM: Stable Persona-driven Agent Simulation for Multi-turn Dialogue Generation

用語言模型生成合成對話，是訓練教學輔導、心理支援等系統的重要資料來源

但當兩個語言模型長時間對話後，它們往往會失去自己的角色設定

一個設定為「求助者」的模型可能逐漸說起「輔導員」的話，甚至開始複製對方的措辭

SPASM 專為解決這類長序列對話中的角色漂移問題而設計 👇

----
多輪對話生成面臨三種主要的身份失穩現象：角色漂移、角色混淆，以及回聲效應

角色漂移指的是角色設定在對話推進過程中逐漸偏離初始定義

角色混淆發生在模型不清楚自己應扮演對話中的哪一方

回聲效應是指一個代理逐漸開始模仿另一個代理的用語和風格

這些失穩現象在長對話中會累積，使得生成的合成數據品質大幅下降

----
SPASM 將對話模擬拆解為三個獨立模組，各自聚焦於不同的穩定性環節

第一個模組負責角色創建：透過結構化的 schema 取樣產生角色設定，並驗證其合理性

第二個模組負責對話生成：採用明確的「求助者（Client）」與「回應者（Responder）」框架區分雙方角色

第三個模組負責終止偵測：判斷對話何時已達到自然結束點，避免無意義的延伸

模組化設計讓每個穩定性問題都有對應的處理機制，不需要一個整體式的解決方案

----
SPASM 最核心的技術貢獻是自我中心上下文投影（Egocentric Context Projection，ECP）

ECP 的做法是將對話歷史以視角中立的格式儲存，而非直接存成任何一方的視角

在每次生成前，系統會確定性地將共享歷史投影成當前生成者的自我中心視角

每個代理只從自己的視角看到歷史，從根本上避免了混淆對方語言模式的問題

整個投影過程是確定性的，不需要呼叫語言模型，計算成本極低

----
ECP 無需修改任何模型權重即可生效

對話歷史的視角轉換在推論時即時執行，可插拔於任何現有的對話生成管道

視角中立的儲存格式也保留了完整的歷史記錄，方便後續分析

SPASM 的框架設計對底層語言模型沒有限制，可搭配不同模型使用

對於需要大量高品質多輪合成對話數據的應用場景，SPASM 提供了一套穩定性優先的生成架構
