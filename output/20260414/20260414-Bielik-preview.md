Advancing Polish Language Modeling through Tokenizer Optimization in the Bielik v3 7B and 11B Series

通用多語言模型在波蘭語上的效果遠不如預期，問題根源在分詞器

一個沒有針對波蘭語設計的分詞器，會把一個波蘭語詞切成更多 token，推論成本因此爆增

Bielik v3 系列從根本上解決了這個問題：換掉分詞器，重新設計詞彙表，並重新訓練整個模型

這是語言特化 LLM 開發的一個值得參考的完整流程案例 👇

----
通用多語言分詞器的設計初衷是讓同一個詞彙表服務盡可能多的語言

這種做法在覆蓋廣度上有優勢，但對形態豐富的語言往往效率低下

波蘭語是一種高度屈折語，詞形變化極為豐富，同一個詞根可以衍生出大量不同的詞形

通用分詞器在面對這種語言時，往往需要更多 token 才能表示同等語義的內容

更高的 token 數量直接導致推論成本上升、有效上下文視窗縮短

----
Bielik v3 將分詞器從基於 Mistral 的通用設計換成了波蘭語專屬詞彙表

專屬詞彙表的核心目標是降低生育率（fertility ratio），讓同等文字內容用更少的 token 表示

更低的生育率直接轉化為更短的序列長度，推論效率因此提升

有效上下文視窗在 token 數量相同的前提下，能覆蓋更長的波蘭語文本

這對於需要處理長文件的應用場景來說是顯著的實用改善

----
換掉分詞器後面臨的挑戰是如何初始化新詞彙的嵌入向量

Bielik v3 採用了基於 FOCUS 的嵌入初始化方法來處理這個問題

FOCUS 利用舊詞彙表與新詞彙表之間的語義關聯，為新 token 生成合理的初始向量

這比隨機初始化更能保留模型在預訓練中習得的語言知識

合理的初始化讓後續的預訓練能更快收斂，減少從頭學習的需求

----
預訓練採用多階段課程式設計，讓模型逐步適應新詞彙表

多階段設計允許在不同階段調整資料分布和訓練目標，提升學習效率

預訓練完成後，模型還需要經過後訓練對齊

後訓練涵蓋監督微調（SFT）、直接偏好優化（DPO）和基於群體相對策略優化（GRPO）的強化學習

這三個步驟分別對應指令跟隨、偏好對齊和策略最佳化三個不同目標
