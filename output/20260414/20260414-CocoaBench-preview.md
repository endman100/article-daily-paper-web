CocoaBench: Evaluating Unified Digital Agents in the Wild

LLM agent 現在能寫程式、做研究、操作 GUI，各種能力都越來越強

但大多數評估測試都是把這些能力分開來考，各考各的

現實中的任務卻往往需要 agent 同時整合視覺、搜尋和程式碼能力才能完成

這個落差讓我們很難知道，整合型的 agent 到底在真實場景中夠不夠用

CocoaBench 就是為了填補這個缺口而生的 👇

----
CocoaBench 是一個專為統一數位 agent 設計的評測基準

裡面的任務全部由人工設計，都是需要長時間推進、完整解決的複雜問題

每個任務只給 agent 一句指令，以及一個針對最終輸出結果的自動評估函式

這種設計讓評估過程可靠且好擴展，不會受到特定 agent 框架的限制

任何基礎架構下執行的 agent 都能直接拿 CocoaBench 來測，評分標準統一

----
CocoaBench 裡的任務需要 agent 靈活組合三種核心能力

第一是視覺理解，agent 要能看懂圖片、介面截圖或文件內容

第二是資訊搜尋，agent 必須主動去網路或資料庫找需要的資料

第三是程式碼執行，agent 要能寫程式、跑程式來完成計算或操作任務

這三種能力要在同一個任務流程中靈活切換，才算真正的統一型 agent

----
為了讓不同模型骨幹的比較更公平，研究團隊同時推出了 CocoaAgent

CocoaAgent 是一個輕量級的共用 scaffold，設計上盡量保持簡單

它的存在不是為了追求最高分，而是提供一個受控的比較環境

不同的大型語言模型可以直接接上這個 scaffold 來跑 CocoaBench

這樣研究者就能清楚看到，同樣的架構下換不同模型會帶來多大的差異

----
實驗結果相當能說明問題：目前最好的系統離穩定可靠還有不小的距離

即使是表現最好的模型組合，成功率也遠未達到讓人放心使用的水準

這不是單一能力不夠強的問題，而是多種能力組合起來就出現明顯的瓶頸

agent 單獨寫程式可能很行，但同時需要搜尋資訊和視覺解析時就開始出錯

這說明現有的 agent 評估框架確實低估了真實任務的困難度

----
研究團隊針對失敗案例做了詳細的錯誤分析，歸納出三個主要的改進空間

第一是推理與規劃，agent 面對長流程任務時容易迷失，不知道下一步該做什麼

第二是工具使用與執行，agent 常常用錯工具，或調用方式有誤導致執行失敗

第三是視覺定位，agent 經常無法準確識別畫面上的元素或理解視覺內容

這三個方向的弱點相互疊加，讓整體成功率遠低於各個能力單獨測試的結果

----
CocoaBench 的出現讓社群有了一個更真實、更有挑戰性的評估標準

過去那種把能力切開來分測的方式，會讓 agent 的表現看起來比實際更好

整合型的評測才能真正反映 agent 在複雜真實任務中的實際水準

這份研究也提醒了開發者，打造真正好用的 agent 需要同時攻克多個技術難關
